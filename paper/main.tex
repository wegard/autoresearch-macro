\begin{abstract}
We test whether LLM-guided search can improve pseudo-real-time forecasts of the Norwegian macroeconomy by choosing covariates, transformations, and fine-tuning settings for Chronos-2. An LLM agent evaluates candidate pipelines on a rolling 2006--2015 validation window. It selects oil prices, the policy rate, US inflation, and the NOK/EUR exchange rate, reducing validation MASE by 6.8\% relative to zero-shot Chronos-2. Yet these gains do not generalize to 2016--2025, which includes the pandemic and post-2022 inflation surge: zero-shot Chronos-2 proves more robust than the agent-tuned pipeline and classical baselines. Ablation shows the search overfits to covariate relationships that fail under regime change. Agent-guided search can find economically interpretable pipelines, but its gains are fragile in non-stationary macroeconomic environments.

\bigskip
\noindent\textbf{Keywords:} macroeconomic forecasting, foundation models, automated machine learning, time series, Norway

\noindent\textbf{JEL codes:} C53, C55, E37
\end{abstract}

\section{Introduction}
\label{sec:introduction}
% ===================================================================

Time series foundation models---large neural networks pretrained on billions of observations from diverse domains---have rapidly entered the forecaster's toolkit. Models such as Chronos \citep{ansari2024chronos,ansari2025chronos2}, TimesFM \citep{das2024decoder}, and MOIRAI \citep{woo2024unified} promise accurate zero-shot forecasts without task-specific training, raising the question of whether they can match or exceed classical methods in rigorous macroeconomic applications.

We investigate this question for Norway---a small open oil-exporting economy with excellent data availability \citep{bjornland2019dutch,aastveit2023inflation}. We study Norway rather than the United States because most economies are small and open, and results from a large, relatively closed economy need not generalize. Because Norwegian macro data are unlikely to appear in TSFM pretraining corpora, the zero-shot exercise is a cleaner test of out-of-distribution generalization. We forecast four target variables---CPI inflation, industrial production, retail sales, and unemployment---at horizons of 1, 3, 6, and 12 months ahead, using a rolling pseudo-real-time evaluation protocol that respects publication lags.

We further investigate whether an \emph{automated search procedure} can improve the forecasting pipeline. Inspired by \citet{karpathy2026autoresearch}, we design an outer loop in which a large language model (Claude Sonnet) proposes pipeline configurations---covariates, transformations, context window, and fine-tuning settings---evaluated against a rolling validation metric. The agent may select from 14 macroeconomic covariates and adjust fine-tuning hyperparameters, but cannot modify the model architecture.

Our main findings are threefold. First, zero-shot Chronos-2 is competitive with ARIMA at short horizons and more robust to the structural breaks of the test period. Second, the search discovers an interpretable pipeline---oil prices, the policy rate, US inflation, and the NOK/EUR exchange rate---that improves validation-era MASE by 6.8\%. Third, this improvement does not generalize: the agent-tuned configuration overfits to covariate relationships that break down under the regime changes of 2016--2025.

This overfitting result is itself a key contribution. It demonstrates a fundamental tension in automated pipeline optimization for macroeconomic forecasting: the covariate relationships that an agent discovers as informative during stable periods may become misleading during crises. Subperiod analysis confirms that the agent-tuned configuration performs well pre-COVID but deteriorates sharply during and after the pandemic, while zero-shot Chronos-2 degrades more gracefully.

\section{Related literature}
\label{sec:literature}
% ===================================================================

This paper connects three literatures: macro forecasting, time-series foundation models, and agentic AutoML.

\subsection{Macroeconomic forecasting}
\label{sec:lit_macro}

Data-rich macro forecasting has progressed from dynamic factor models \citep{stock2002macroeconomic,stock2002forecasting,forni2005generalized} through Bayesian VARs with shrinkage priors \citep{banbura2010large,koop2013large} to machine learning methods that capture non-linearities \citep{coulombe2022machine,babii2024econometrics}. Real-time evaluation must contend with the ragged-edge problem of asynchronous publication \citep{ghysels2004predicting,giannone2008nowcasting}; our pseudo-real-time protocol builds directly on this tradition, using expanding windows and \citet{diebold2002comparing} tests as standard \citep{clark2013advances}.

Forecasting Norway specifically involves exposure to oil shocks, exchange rate volatility, and collinear domestic indicators \citep{bjornland2019dutch,aastveit2023inflation}. \citet{larsen2021components} finds predictive content in text-based uncertainty measures for the Norwegian business cycle, motivating our interest in automated covariate discovery.

\subsection{Foundation models for time series}
\label{sec:lit_tsfm}

Time series foundation models (TSFMs) are pretrained on massive, heterogeneous corpora to enable zero-shot inference on unseen series. Chronos \citep{ansari2024chronos} treats forecasting as language modeling via value quantization; its successor Chronos-2 \citep{ansari2025chronos2} adds group attention that alternates between within-series and cross-series patterns, enabling in-context learning from covariates without architectural changes. Chronos-2 is a natural backbone for our pipeline: the agent's task is to discover which covariates to present to the model. Alternative TSFMs include TimesFM \citep{das2024decoder,das2025incontext}, Lag-Llama \citep{rasul2023lagllama}, and MOIRAI \citep{woo2024unified,liu2025moirai2}; concerns about pre-training data contamination are documented by \citet{aksu2024benchmarking}.

The tension between zero-shot inference and task-specific fine-tuning is central to applied TSFM work \citep{rasul2023lagllama}. Our research interrogates this tradeoff by decomposing forecast gains into the foundation model itself, economy-specific adaptation, and the additional value of agent-guided search.

\subsection{Automated machine learning and feature engineering}
\label{sec:lit_automl}

Traditional AutoML frameworks \citep{feurer2015automl,erickson2020autogluon,shchur2023autogluon} employ Bayesian optimization and meta-learning but operate within rigidly defined search spaces and cannot incorporate domain knowledge. A newer paradigm uses LLMs as meta-optimizers that reason about results and autonomously modify experimental configurations \citep{lu2024aiscientist,yamada2025ai,cheng2026agentic,dong2025dataforge}.

Our approach builds on Karpathy's ``autoresearch'' framework \citep{karpathy2026autoresearch}, which uses a ratchet loop with fixed data preparation, a mutable training script, and a natural-language specification of the objective. This has been adopted across disciplines \citep{oconnell2026signal,weidener2026rethinking,liu2026autoresearchclaw} but not yet in macroeconomic forecasting. We adapt it by constraining the search to data representations, covariate selection, and fine-tuning---never model architecture---with pseudo-real-time evaluation governed by rolling forecast-origin validation loss.

\section{Methodology}
\label{sec:methodology}
% ===================================================================

\subsection{Foundation model: Chronos-2}
\label{sec:chronos}

We use Chronos-2 \citep{ansari2025chronos2} as the foundation model backbone, accessed via the AutoGluon TimeSeries interface \citep{shchur2023autogluon}. The model has 120 million parameters and supports a maximum context length of 8,192 time steps. Its key architectural innovation is a group attention mechanism that alternates between time attention (within a single series) and group attention (across series in a batch), enabling in-context learning from covariates without requiring explicit covariate-specific input channels.

Unlike earlier Chronos-Bolt models, which produce deterministic point forecasts and ignore additional columns in the input data, Chronos-2 natively integrates both past covariates and known future covariates during inference. In our implementation, all macroeconomic covariates are provided as \emph{past covariates only}: the model receives historical values up to the forecast origin but no future values. This is critical for pseudo-real-time validity---future values of oil prices, exchange rates, and inflation are not known at the forecast origin. The covariates are included as additional columns in the AutoGluon TimeSeriesDataFrame; no \texttt{known\_covariates} are passed to the predict call.

\paragraph{Fine-tuning.} Chronos-2 supports parameter-efficient adaptation via Low-Rank Adaptation (LoRA), which targets the query, key, value, and output projections of the self-attention layers plus the output patch embedding. The default configuration uses rank $r = 8$ and $\alpha = 16$. In our setup, fine-tuning is performed once on the data available at the first forecast origin, and the adapted model is reused for all subsequent origins. This fit-once architecture means the fine-tuned model may overfit to early-period patterns---a limitation we return to in Section~\ref{sec:discussion}.

\subsection{Agentic search procedure}
\label{sec:search}

The search is orchestrated by an outer loop that uses a large language model (Claude Sonnet, via the Anthropic API) to propose pipeline configurations. The loop follows a ratchet design inspired by \citet{karpathy2026autoresearch}:

\begin{algorithm}[H]
\caption{LLM-guided pipeline search}
\label{alg:search}
\begin{algorithmic}[1]
\State Establish baseline: evaluate zero-shot Chronos-2 on all 120 validation origins
\For{$i = 1, 2, \ldots, N$}
    \State LLM reads: task description, search space, current best config, iteration history
    \State LLM proposes a JSON configuration (covariates, transforms, context length, fine-tuning)
    \State \textbf{Quick evaluation:} run on 20 subsampled origins ($\sim$30 seconds)
    \If{quick score $<$ current best}
        \State \textbf{Full evaluation:} run on all 120 origins ($\sim$3 minutes)
        \If{full score $<$ current best}
            \State \textbf{Accept:} update best configuration and score
        \Else
            \State \textbf{Reject}
        \EndIf
    \Else
        \State \textbf{Reject}
    \EndIf
    \State Log iteration (config, scores, status) to experiment log
\EndFor
\end{algorithmic}
\end{algorithm}

The two-phase evaluation balances iteration speed with evaluation fidelity. The baseline and all accept/reject decisions use the full 120-origin evaluation to ensure fair comparisons. The primary metric is the mean absolute scaled error (MASE), averaged across all four target variables and all four forecast horizons.

\paragraph{Prompt design.} The LLM receives a natural-language specification (\texttt{program.md}) containing the task description, constraints, available search dimensions, domain knowledge about the Norwegian economy (e.g., ``Norway is an oil-exporting economy; Brent crude and exchange rates are highly relevant''), and strategy suggestions. The agent returns a JSON object specifying only the fields it wishes to change from the current best configuration.

\section{Data and evaluation protocol}
\label{sec:data}
% ===================================================================

\subsection{Norwegian macroeconomic panel}
\label{sec:panel}

We construct a monthly panel of 18 macroeconomic variables from three sources: Statistics Norway (SSB), the Federal Reserve Economic Data (FRED), and Norges Bank. Table~\ref{tab:variables} in Appendix~\ref{app:data} provides the full variable catalog. The four target variables are CPI inflation (12-month rate of change), industrial production (manufacturing, seasonally adjusted), retail sales (volume index, seasonally adjusted), and the unemployment rate (LFS, seasonally adjusted). The 14 covariates span domestic indicators (house prices, credit, exports, imports, the policy rate, exchange rates) and global variables (Brent crude oil, NASDAQ, the federal funds rate, US CPI, Euro area GDP, VIX, global economic policy uncertainty).

\subsection{Pseudo-real-time discipline}
\label{sec:realtime}

At each forecast origin $t$, we restrict the information set to data that would have been publicly available on date $t$. For a monthly observation indexed at month-end $m$, the observation is available when
\begin{equation}
    \text{month\_end}(m) + \text{publication\_lag} \leq t.
\end{equation}
Publication lags range from 0 days (the policy rate, known in real time) to 90 days (Euro area GDP). Daily series (exchange rates, oil prices, VIX) are aggregated to monthly averages; quarterly series (house prices, Euro area GDP) are forward-filled. We use latest-vintage data throughout but restrict availability by publication lag. This is pseudo-real-time, not true real-time with data vintages.\footnote{Investigating the additional effect of data revisions is left for future work.}

\subsection{Evaluation design}
\label{sec:evaluation}

We use a rolling expanding-window design. For each forecast origin $t$, the model receives all data available up to $t$ and produces point forecasts at horizons $h \in \{1, 3, 6, 12\}$ months ahead for each target variable.

\paragraph{Eras.} The validation era spans January 2006 to December 2015 (120 monthly origins), used for search and model selection. The test era spans January 2016 to March 2025 (111 origins), evaluated with the pipeline frozen from the search phase. We further decompose the test era into three subperiods: pre-COVID (2016--2019), COVID (2020--2021), and post-COVID (2022--2025).

\paragraph{Metrics.} We report root mean squared error (RMSE), mean absolute error (MAE), and mean absolute scaled error (MASE). The MASE is defined as $\text{MASE}_h = \text{MAE}_h / \text{MAE}_h^{\text{RW}}$, where $\text{MAE}_h^{\text{RW}}$ is the MAE of the $h$-step-ahead random walk forecast at each origin. Under this definition, the random walk scores exactly 1.0 by construction, and values below 1 indicate improvement. MASE is the primary metric throughout this paper because it is scale-independent across target variables with different units (percentage rates, index levels).

\subsection{Baselines}
\label{sec:baselines}

We evaluate seven baselines spanning univariate and multivariate approaches:

\paragraph{Univariate baselines.}
\begin{enumerate}[noitemsep]
    \item \textbf{Random walk:} $\hat{y}_{t+h|t} = y_t$ (last observed value).
    \item \textbf{Seasonal naive:} $\hat{y}_{t+h|t} = y_{t+h-12}$.
    \item \textbf{AR($p$):} Direct autoregressive regression with lag order selected by BIC over $p \in \{1, \ldots, 12\}$.
    \item \textbf{ARIMA:} Order selected by AIC over $p \in \{0,1,2,3\}$, $d \in \{0,1\}$, $q \in \{0,1,2\}$.
    \item \textbf{ETS:} Exponential smoothing with automatic trend/seasonal selection by AIC.
\end{enumerate}

\paragraph{Multivariate baselines.} To ensure a fair comparison with the covariate-augmented Chronos-2, we include two multivariate methods that access the same information set:
\begin{enumerate}[noitemsep,resume]
    \item \textbf{VAR:} Vector autoregression on the target variable plus the four covariates selected by the search agent (Brent crude, policy rate, US CPI, NOK/EUR). Lag order selected by BIC, capped at 6.
    \item \textbf{Factor model:} Principal components extracted from all 14 covariates, used as regressors in direct $h$-step-ahead forecasting equations alongside autoregressive lags of the target.
\end{enumerate}
All baselines are re-estimated at each forecast origin using only data available at that origin, ensuring full pseudo-real-time compliance.

\paragraph{Statistical inference.} We test for equal predictive ability using the \citet{diebold2002comparing} test with Newey-West HAC standard errors to account for serial correlation in multi-step forecast errors. The bandwidth is set to $h - 1$ for $h$-step-ahead forecasts.

Figure~\ref{fig:trajectory} visualizes the search trajectory. The search converged by approximately iteration 45. The remaining iterations proposed configurations that either matched or worsened the best score. Key patterns from the 50 iterations:

\begin{itemize}[noitemsep]
    \item \textbf{Covariates discovered incrementally:} The agent first added oil prices, then the policy rate, then US inflation, and finally the exchange rate---each improving the score marginally.
    \item \textbf{Context window matters:} Truncating to 96 months (8 years) improved over unlimited context, suggesting that very old data introduces noise.
    \item \textbf{Fine-tuning is fragile:} Only very conservative LoRA settings (100 steps, lr $= 5 \times 10^{-6}$) were accepted. All attempts with 500+ steps or higher learning rates degraded performance.
    \item \textbf{Transforms don't help:} Raw covariate levels outperformed log-differences, percentage changes, and standardization.
    \item \textbf{The configuration is economically interpretable:} Oil prices, monetary policy, global inflation, and the trade-weighted exchange rate are precisely the variables a macroeconomist would expect to matter for a small open oil-exporting economy.
\end{itemize}

% ===================================================================
\section{Discussion}
\label{sec:discussion}
% ===================================================================

\paragraph{Zero-shot robustness.}
Zero-shot Chronos-2 is more robust to structural breaks than both the agent-tuned model and classical ARIMA. Pretraining on diverse time series patterns appears to provide implicit regularization against overfitting to specific covariate relationships. For practitioners, zero-shot foundation models may be preferable to tuned pipelines when regime changes are likely.

\paragraph{The overfitting paradox.}
The agent discovers an economically interpretable, statistically superior configuration---yet it fails out of sample. This reflects a fundamental limitation of fixed-window macroeconomic validation rather than flawed agentic reasoning. The policy rate covariate contributes the largest overfitting, consistent with the regime shift from near-zero rates to rapid tightening. Periodic retuning does not help, confirming that the covariate relationships themselves---not stale model weights---are the problem.

\paragraph{Domain knowledge as a double-edged sword.}
The domain-blind experiment shows the LLM discovers meaningful covariates autonomously, but domain hints bias the informed agent toward oil-specific channels and context truncation that drive the most overfitting. Domain knowledge improves validation performance but may increase fragility to regime changes.

\paragraph{Limitations.}
We use a single foundation model; results may differ for TimesFM or MOIRAI. Our pseudo-real-time protocol uses latest-vintage data, not true real-time vintages. We evaluate only point forecasts; density evaluation using CRPS could reveal different patterns. The evidence comes from one country with four targets; generalization to other economies remains to be tested.


% ===================================================================
\section{Conclusion}
\label{sec:conclusion}
% ===================================================================

LLM-guided search discovers an economically interpretable pipeline for Norwegian macro forecasting---oil prices, the policy rate, US inflation, and the exchange rate---that improves validation MASE by 6.8\%. Yet the ablation reveals that every accepted improvement monotonically widens the validation-to-test gap, and the gains do not survive the regime changes of 2016--2025. Zero-shot Chronos-2 proves more robust than both the agent-tuned pipeline and classical baselines.

Two implications follow. First, zero-shot foundation models should be included as standard baselines in macro forecasting---their robustness to structural breaks is a practical advantage. Second, automated pipeline search can identify interpretable configurations, but fixed-window validation in a non-stationary environment risks overfitting to a particular regime. The policy-rate shift between eras illustrates this: relationships that help in stable periods fail in crises. Future work should explore rolling validation windows, per-variable search, and extension to multiple foundation models and economies.

\section{Agent experiment log}
\label{app:experiment_log}

The full 50-iteration experiment log is available in the project repository (\texttt{results/search\_log.jsonl}). Out of 50 iterations, 6 were accepted (12\% acceptance rate). The search converged by approximately iteration 45. Common rejected proposals included: adding more than 4 covariates (consistently worsened scores), aggressive fine-tuning (500+ steps or learning rate $> 10^{-5}$), and data transformations on covariates. The agent's reasoning was generally sensible---it systematically tested individual covariates, tried combinations, explored fine-tuning, and adjusted context lengths.

For comparison, a 50-iteration random search achieved a 2\% acceptance rate with a single accepted configuration at iteration 1 (validation MASE 1.799). The random search's best configuration used 5 covariates (house prices, Euro area GDP, global EPU, imports, policy rate) with mixed transformations---economically less coherent than the LLM's incremental discovery.

\noindent All dependencies are pinned in a \texttt{uv.lock} file. The project uses \texttt{uv} for dependency management. Hardware: NVIDIA GeForce RTX 4090 (24\,GB VRAM).

\subsection*{Foundation model}

Chronos-2 is loaded via AutoGluon using the model identifier \texttt{amazon/chronos-2} and the \texttt{"Chronos-2"} model key. The model has 120M parameters. For zero-shot inference, the model weights are loaded directly from the HuggingFace Hub. For LoRA fine-tuning, the target modules are the self-attention Q, K, V, O projections and the output patch embedding, with rank $r = 8$ and $\alpha = 16$. Point forecasts are extracted from the \texttt{mean} column of the AutoGluon prediction output.

\subsection*{LLM search agent}

The search agent uses Claude Sonnet (\texttt{claude-sonnet-4-20250514}) via the Anthropic Messages API with \texttt{max\_tokens=1024} and default temperature. The system prompt (\texttt{program.md}) contains the task description, search space constraints, available covariates, and---in the informed variant---domain knowledge about the Norwegian economy. The full prompt text is available in the project repository. The domain-blind variant (\texttt{configs/program\_blind.md}) removes all country-specific hints.

At each iteration, the agent receives the current best configuration, the MASE score, and a history of the most recent 15 iterations (config, score, status). The agent returns a JSON object specifying only the fields to change. Responses are parsed by extracting the first JSON object found, with fallback handling for markdown code blocks.

\subsection*{Data retrieval}

All data sources are publicly available:
\begin{itemize}[noitemsep]
    \item \textbf{Statistics Norway (SSB):} JSON-stat2 API at \texttt{data.ssb.no/api/v0}. No authentication required. Table IDs and content codes are listed in Table~\ref{tab:variables}.
    \item \textbf{Norges Bank:} SDMX-CSV API at \texttt{data.norges-bank.no/api/data}. No authentication.
    \item \textbf{FRED:} Via the \texttt{fredapi} Python client. Requires a free API key from \texttt{fred.stlouisfed.org}.
    \item \textbf{Anthropic API:} Required for the LLM search agent. Not required for baselines or zero-shot evaluation.
\end{itemize}

\noindent Data is cached locally after the first download. Publication lags are configured in \texttt{configs/publication\_lags.yml}. Daily series are aggregated to monthly averages; quarterly series are forward-filled. All covariates are provided as past-only values (no future covariate leakage).

\subsection*{Evaluation protocol}

Forecast origins are generated at monthly frequency. Validation era: January 2006 through December 2015 (120 origins). Test era: January 2016 through March 2025 (111 origins). At each origin, the information set is restricted by publication lags. Quick evaluation during search uses 20 origins subsampled with seed 42; full evaluation uses all origins. The baseline is always scored on all origins to ensure fair comparisons.

\subsection*{Random seeds and determinism}

Origin subsampling uses \texttt{numpy.random.default\_rng(42)}. AutoGluon and PyTorch use default seeds (not explicitly controlled). Chronos-2 produces deterministic point forecasts via a single forward pass. The LLM search agent is non-deterministic due to the stochastic nature of the Claude API; repeated runs may discover different configurations.
